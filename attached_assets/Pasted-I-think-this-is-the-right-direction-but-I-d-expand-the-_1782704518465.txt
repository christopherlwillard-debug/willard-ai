I think this is the right direction, but I'd expand the scope just a little so you don't leave "Immich-shaped holes" in the architecture.

Instead of making this just a cleanup task, make it the point where the app officially becomes self-contained.

I'd add these items:

1. Rename Media components

If you still have files named things like:

immich-service.ts
immich-client.ts
immich-sync.ts

don't just delete them—replace them with your own terminology.

For example:

library-engine.ts
thumbnail-engine.ts
media-index.ts
library-service.ts

That keeps the architecture consistent.

2. Remove Immich terminology from the UI

Search for strings like:

"Rescan Immich"
"Sync to Immich"
"Immich Library"
"Immich Server"

Replace them with:

Scan Library
Refresh Library
Library Engine
Media Library

Users should never know Immich was ever part of the project.

3. Remove Docker assumptions

I'd broaden this slightly.

Don't just remove the word "Docker."

Also remove any code that assumes:

/data/upload
/data/library
Linux-only paths
Docker volume paths

Everything should now use the configured library path:

Library Path

Z:\

or

\\MYCLOUD-G1S9ZJ\Wilzard

No hard-coded filesystem assumptions.

6. Rename the task

Instead of:

Remove Immich & Docker references

I'd call it:

Transition to Native Library Engine

That better reflects what you're doing.

I'd also update the Definition of Done

Add these requirements:

No runtime dependency on Immich or Docker remains.
The application can perform scanning, metadata extraction, thumbnail generation, and library browsing using only the configured filesystem path.
All user-facing terminology refers to the Library Engine or Media Library rather than Immich.
All filesystem operations use the configured Library Location instead of hard-coded paths.
This is actually a major milestone

My Cloud Home (NAS)
        │
        ▼
