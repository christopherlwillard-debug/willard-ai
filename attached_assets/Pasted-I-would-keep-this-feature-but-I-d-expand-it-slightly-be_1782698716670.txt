#46 - Enrich Media Index with Metadata Extraction

What & Why

The Media Center should extract metadata from media files during indexing so users can browse, sort, search, organize, and filter without rescanning files later.

Metadata should become part of the Media Index and power future features like timeline view, map view, duplicate detection, smart collections, and advanced search.

This scan only reads metadata. Original files are never modified.

One thing I'd add

I'd also tell Replit not to scan the full EXIF every time.

Use this logic:

If the file is new → extract metadata.
If the file's size or modified date changed → re-extract metadata.
Otherwise → skip it.

That keeps rescans fast, even with very large photo libraries. This becomes important when you're indexing hundreds of thousands of files on your NAS.
