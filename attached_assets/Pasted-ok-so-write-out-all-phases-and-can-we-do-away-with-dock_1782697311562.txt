Yes. I actually think this is the right decision.

You've spent a lot of time trying to make Immich fit your architecture, when what you really want is your own Media Center. Immich can always be added later if you decide its AI features are worth it, but I would remove it completely from the roadmap for now.

Here's the roadmap I'd give Replit.

Media Center Roadmap (Docker & Immich Free)
Philosophy

The Media Center owns the media library.

The NAS is the single source of truth.

The app never copies media onto the computer unless the user explicitly downloads it.

No Docker.

No Immich.

No PostgreSQL.

No AI dependencies.

Everything works directly from the NAS.

Phase 1 – Foundation
Storage Setup

The user selects the root media folder.

Example:

Z:\MediaCenter
Z:\Photos

The app remembers this location.

Build a scanner that:

Walks every folder
Reads every supported media file
Creates a media index
Detects deleted files

Detects modified files

Phase 10 – Optional AI (Future)

AI is completely optional.

Possible providers:

Immich
Google Vision
OpenAI
Local AI

Features:

Automatic Tag Suggestions

The Media Center never depends on AI.

AI only enhances existing features.

Core Principles
NAS is always the master copy.
Original files are never altered without user approval.
The Media Center owns the media database.
The Media Center generates its own thumbnails.
Every operation works without Docker.
Every operation works without Immich.
AI is an optional enhancement, never a requirement.

I actually think this is a much stronger product.

You're no longer building "something that works because Immich exists." You're building your own digital asset manager.

That also gives you freedom in the future. If one day you buy a mini PC and decide you want AI search, you can plug it in. If a better AI tool than Immich comes along, you can integrate that instead. None of your core architecture has to change because the Media Center remains the system that owns the library, the database, and the workflow.
