I actually think your instinct is right. I wouldn't optimize images the way your current app is suggesting.

For photos (JPEG, JPG, HEIC)

I would not convert them to WebP as the default.

Instead I'd make the optimizer support profiles.

Profile	Target	Quality	Use Case
Archive (Recommended)	JPEG XL (future) or JPEG	Visually lossless	Long-term storage
High Quality JPEG	JPEG	95-98%	Best compatibility
Balanced JPEG	JPEG	90-92%	20-40% smaller with almost invisible difference
Maximum Compression	AVIF/WebP	Adjustable	Only if user explicitly wants it

For your app, I'd make High Quality JPEG the default.

Better than just changing format

Instead of converting every JPG to another format, I'd optimize the existing JPEG.

Things like:

Progressive JPEG
Remove EXIF thumbnails
Strip duplicate metadata
Optimize Huffman tables
Recompress only if smaller

Many JPEGs shrink 10–30% with zero visual quality loss.

My biggest recommendation

I would change Optimize from "format conversion" into an Optimization Engine.

Instead of trying to convert everything to WebP, it should recommend the best action per file type:

JPEG → Optimize JPEG (lossless by default).
PNG → Lossless PNG optimization.
HEIC → Leave as-is unless the user requests otherwise.
AVI/MPG → Re-encode to H.265 with configurable quality (where large savings are realistic).
MOV → Analyze first. If it's already H.264/H.265, leave it; if it's an older codec, offer conversion.
MP4 → Don't convert if it's already efficient—simply mark it "Optimal."

That approach preserves quality, avoids unnecessary conversions, and gives users confidence that the optimizer is making intelligent decisions rather than blindly changing formats. I think it also fits the vision you've described for Willard AI: an application that understands your media and recommends the right optimization for each file, instead of applying one-size-fits-all rules.
