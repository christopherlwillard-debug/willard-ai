One feature I think would make Willard AI unique

Don't make optimization decisions based on file extension. Make them based on codec and file characteristics.

H.264 (leave it alone)

If you've truly decided you'll never edit the RAW files again, then converting them can make sense—but I'd still be careful about what you convert them to.

My recommendation
📷 RAW → High-Quality JPEG (Recommended)
Quality: 98–100
Chroma subsampling: 4:4:4 if supported
Embed ICC color profile
Keep EXIF metadata
Keep GPS/date/camera/lens information
Auto-rotate using EXIF
Progressive JPEG
Optimize Huffman tables

This gives you a file that's:

Much smaller than RAW (often 70–90% smaller)
Universally supported
Excellent visual quality
Likely to remain readable for decades
Don't just batch convert

RAW files have no "finished" look until they're developed. The converter should:

Apply the camera's embedded color profile.
Use high-quality demosaicing.
Preserve white balance.
Preserve highlights and shadows as well as possible.
Export at the original resolution.

Otherwise, some RAW photos can look flat or different from what your camera showed.

I would make Willard AI use a two-stage process:

Convert the RAW.
Verify:
JPEG was created successfully.
Resolution matches the RAW.
Metadata is preserved.

Only then delete the RAW (if the user chose that option).

That protects against losing irreplaceable photos because of a failed conversion.
