This is one of the most important performance features you'll build. I would expand it from "skip hashing" into a full Incremental Scan Engine because that's what users expect from a modern media manager.

#7 - Incremental Media Scanner (Skip Unchanged Files)

What & Why

The Media Center should never perform unnecessary work.

During a scan, files that have not changed since the previous scan should not be rehashed, have their metadata re-extracted, or have their thumbnails regenerated.

Instead, the scanner should intelligently detect which files actually changed and process only those.

This allows repeat scans of very large NAS libraries to complete in seconds rather than hours.

The NAS remains the source of truth while the Media Index stays synchronized efficiently.

Performance Goals
Never hash an unchanged file.
Never regenerate an existing thumbnail.
Never reread EXIF for unchanged media.
Only access files that actually changed.
Scans should scale to hundreds of thousands of files.
